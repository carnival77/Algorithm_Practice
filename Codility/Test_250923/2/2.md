# 🧩 **Task 2 — Grid Symmetry (격자 대칭 문제)**

## 📘 **문제 설명**

N×M 크기의 격자가 주어진다.
각 칸은 **검정색(`B`)** 또는 **흰색(`W`)** 중 하나로 색칠되어 있다.

한 **행(row)** 또는 **열(column)** 이
앞뒤로 읽어도 같은 문자열이면 **대칭(symmetric)** 이라 한다.
예를 들어,

* `"BWWBWBW"` → 대칭 아님 ❌
* `"BWWWWB"` → 대칭 ⭕

이제 당신의 목표는,
**모든 행과 열이 대칭이 되도록 최소한의 칸 색깔을 바꾸는 것**이다.

한 번의 움직임(move)으로는
격자 내 임의의 칸 하나의 색(`B` ↔ `W`)을 뒤집을 수 있다.

---

## 🧩 **입력 형식**

* `grid`: N개의 문자열로 구성된 리스트 (각 문자열의 길이는 M)
* 각 문자는 `'B'` 또는 `'W'`
* 1 ≤ N, M ≤ 100

---

## 🧮 **출력 형식**

모든 행과 열을 대칭으로 만들기 위한 **최소 움직임 수**를 출력한다.

---

## 🧠 **예시**

### ✅ 예제 1

```
grid = [
  "BBWBB",
  "BWBBW",
  "BBWWB"
]
```

**결과:** 3

**설명:**
총 3칸을 뒤집으면, 모든 행과 열이 대칭이 된다.

---

### ✅ 예제 2

```
grid = [
  "BWB",
  "WBB",
  "WBW"
]
```

**결과:** 4

**설명:**
4번 색을 바꾸면 다음과 같이 모든 행·열이 좌우 대칭 구조를 만족한다.

---

## 💡 **핵심 아이디어**

1️⃣ **대칭 그룹(Orbit) 단위 처리**

* 한 칸 `(i, j)`는 다음 세 칸과 대칭 관계를 가짐:

  ```
  (i, j), (i, M-j-1), (N-i-1, j), (N-i-1, M-j-1)
  ```
* 즉, 이 네 칸이 서로 같은 색이면
  그 행과 열은 해당 위치에서 대칭을 유지.

2️⃣ **그룹별 최소 변경 계산**

* 각 그룹에 대해 `'B'` 와 `'W'` 개수를 세고
  → `min(B, W)` 만큼만 색을 바꾸면 모두 동일하게 만들 수 있음.

3️⃣ **홀수 행·열 처리**

* 홀수인 경우 중앙 행/열은 자기 자신만 대칭이므로
  중복 없이 한 번만 고려.

---

## ⚙️ **의사코드**

```python
def solution(grid):
    n, m = len(grid), len(grid[0])
    ans = 0
    for i in range((n + 1)//2):
        for j in range((m + 1)//2):
            cells = [
                grid[i][j],
                grid[i][m - j - 1],
                grid[n - i - 1][j],
                grid[n - i - 1][m - j - 1]
            ]
            b = cells.count('B')
            w = 4 - b
            ans += min(b, w)
    return ans
```

---

## 📚 **핵심 개념 요약**

| 핵심 요소        | 설명                                  |
| ------------ | ----------------------------------- |
| **대칭성**      | (i, j) ↔ (N−i−1, M−j−1) 관계          |
| **최소 변경 원리** | 다수결(Majority rule): 그룹 내 가장 많은 색 유지 |
| **시간복잡도**    | O(N×M)                              |
| **메모리복잡도**   | O(1)                                |
| **핵심 알고리즘**  | 구현 + 그리디(대칭 그룹 단위 처리)               |
